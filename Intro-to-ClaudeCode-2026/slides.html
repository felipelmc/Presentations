Do chat ao agente

Você provavelmente já usa IA 💬

  • Interface de chat: você digita, o modelo responde
  • Cada conversa começa do zero: sem memória de sessões anteriores, sem acesso ao seu ambiente de trabalho
  • Interações transacionais: úteis para tarefas pontuais, mas sem continuidade

O chat tem limites para pesquisa

  • Sem acesso ao seu ambiente de trabalho (arquivos, código, terminal)
  • Sem memória entre sessões
  • Coletar dados, ajustar modelos, documentar decisões, comunicar achados: você coordena tudo manualmente

O que é um agente de IA? 🤖

Em vez de responder a perguntas pontuais, o agente planeja ações, executa código e itera sobre tarefas complexas com supervisão mínima.

Ferramentas: lê e escreve arquivos · executa código · chama APIs · navega na web

Claude Code 🖥️

  • CLI que roda diretamente no terminal
  • Lê e escreve arquivos, executa código, acessa o git e o shell
  • Trata-se do caso mais desenvolvido de agente de IA para uso em pesquisa

CLAUDE.md: as regras do jogo 📋

Arquivo de instruções persistentes, lido automaticamente a cada sessão


O que colocar:

  • Contexto do projeto e objetivos
  • Convenções e restrições analíticas
  • Decisões já tomadas (e por quê)

É a memória permanente do agente no projeto – o que sobrevive ao /clear

Skills: comandos customizados ⚡

/nome-da-skill executa um prompt reutilizável, portável entre projetos

/voice           # reescreve texto no seu estilo
/document-notes  # organiza anotações brutas
/review          # revisa código ou argumento

Na prática, é o mecanismo de ensinar o agente a executar tarefas no seu vocabulário e com suas convenções

Demo 1: pesquisa legislativa aplicada 🏛️

A ideia é simples: usar o agente para construir, do zero, um protótipo de pesquisa empírica sobre o Legislativo federal.

Roteiro ao vivo:

  1. Coletar proposições em educação via API da Câmara dos Deputados
  2. Embeddar os textos (representação semântica vetorial)
  3. Produzir visualizações: clusters temáticos, distribuição temporal, temas latentes

Demo 2: skill de “voice” 🖊️

Você tem um estilo de escrita. O modelo pode aprendê-lo.


Como criar: (i) reunir writing samples seus (artigos, e-mails, textos); (ii) criar o arquivo de skill /voice


Uso: pedir ao Claude que revise ou reescreva textos no seu estilo

Demo 3: documentando anotações 📚

Skill criada para o meu próprio workflow


  • Input: anotações brutas de disciplinas
  • Output: arquivos organizados em repositório público


A skill /document-notes lê, estrutura e abre o PR; você só anota

Demo 4: site pessoal a partir do currículo 🌐

  • Input: currículo em PDF ou texto simples
  • Output: site HTML/CSS completo e responsivo
  • Publicável no GitHub Pages sem configuração manual

O mesmo raciocínio vale para páginas de projeto: o gap entre ter um artigo com código e ter uma página sobre ele é menor do que parece – o Claude Code lê o repositório e produz o HTML.

Os limites da delegação ⚠️

  • O agente comete erros; a revisão é parte do workflow, não exceção
  • Sessões longas degradam qualidade – à medida que o contexto enche, as respostas perdem coerência
  • Reprodutibilidade depende de documentar decisões analíticas, não apenas o código gerado
  • O uso de IA na pesquisa científica ainda está sendo estudado – há literatura nova saindo com frequência

Em última instância, o pesquisador responde por tudo que assina – inclusive pelo que o agente produziu.

Quando a supervisão é insubstituível 🔑

  • A interpretação substantiva dos achados
  • As decisões de design: o que medir, como operacionalizar, quais hipóteses testar
  • A validação: o agente sabe se o código roda; você verifica se o achado faz sentido
  • O controle da ferramenta: entender o que foi feito, passo a passo, antes de confiar no resultado
